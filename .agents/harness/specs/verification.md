# Verification Spec

Before saying work is complete:

1. Identify the claim.
2. Identify the command or live scenario that proves it.
3. Run it fresh.
4. Read the output.
5. Confirm the evidence covers the claim.

For performance work, include:

- App page or state.
- Music playback state.
- Harness command.
- CPU average, p95, max.
- CSV path when available.

For visual work, include:

- Screenshot or Computer Use observation.
- What changed visually.
- What was preserved.
