# Documentation Spec

Documentation has three layers:

- L1: `AGENTS.md` and `CLAUDE.md` for project-wide rules and architecture.
- L2: module-level `CLAUDE.md` files for file lists and responsibilities.
- L3: source file headers where the project already uses them.

Update documentation when:

- A module, service boundary, or workflow changes.
- A new harness rule or verification gate is added.
- A new performance trap is verified.
