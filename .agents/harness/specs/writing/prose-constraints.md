# Prose Constraints

## Core Principles

1. **Each sentence must advance.** No rephrasing. If the next sentence says the same thing differently, delete it.
2. **Dig deeper, don't decorate.** When tempted to add a metaphor, ask: does this reveal something new? If not, cut it.
3. **Say what something IS.** No negation framing ("not X but Y"). No slashes in prose ("love/hate"). State directly.
4. **Never assume reader reactions.** Don't write "the reader will feel..." or "this surprises us." Present the material; let reactions happen.
5. **No comparison logic.** Don't structure arguments as "unlike X, Y does Z." Each thing stands on its own terms.

## Red Flags (stop if you catch yourself)

- Stacking 3+ metaphors for the same idea → pick the best one, delete the rest
- "In other words..." → you just rephrased. Delete and move forward.
- "It's as if..." followed by "Or rather..." → indecision. Commit to one image.
- Abstract nouns piling up ("the complexity of the relationship") → make it concrete

## Quality Benchmark

The user's own scrutiny is the benchmark, not AI detection tools.
Alain de Botton (English + Chinese translations) is the go-to exemplar for love/daily life/status.

## During Drafting Phase

- Forward momentum only. Don't revise as you draft.
- If stuck, skip ahead. Come back in revision.
- Raw > polished at this stage. Capture the thought, refine later.

## During Revision Phase

- Cut 20% before adding anything new
- Challenge every metaphor: does it earn its place?
- Read aloud (mentally): rhythm matters as much as meaning
