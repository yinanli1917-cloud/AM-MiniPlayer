# Research Protocol (Writing)

## Principle

Gather material BEFORE drafting. Research disappears on /compact.
Persist findings to files — conversations are ephemeral, files are permanent.

## Process

1. **Define scope** — what questions must be answered before writing?
2. **Search broadly** — WebSearch, WebFetch (with L1→L2 escalation), Apple Notes
3. **Persist each finding** — write to `{task}/research/{topic}.md`
4. **Report paths only** — don't paste full content into chat (wastes context)
5. **Synthesize** — once research files are complete, read them and outline

## Research File Format

```markdown
# {Topic}

**Source:** {URL or note reference}
**Retrieved:** {date}

## Key Points
- ...

## Relevance to Task
- ...

## Direct Quotes (if needed)
> "..."
```

## Boundaries

- Research agent writes ONLY to `research/*.md`
- Research agent does NOT draft prose
- Research agent does NOT edit existing project files
- If research reveals a structural change needed → report it, don't do it
