# Implement Protocol

Use this protocol for code changes.

## Boundaries

- Can edit code and tests inside this repo.
- Cannot run `git push`.
- Cannot modify protected lyric animation or layout without visual parity evidence.
- Cannot claim UX is preserved without inspecting the running app or screenshots.

## Process

1. Read the relevant code and postmortems.
2. Choose the smallest change that addresses the measured issue.
3. Add or update tests when the behavior is testable.
4. Run build and relevant verification.
5. Keep commits separated by concern.
