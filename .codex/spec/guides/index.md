# Codex Spec Guides

Shared project guidance that should be loaded before durable implementation work.

## Pre-Development Checklist

- Read the active task `prd.md`.
- Read any files listed in `implement.jsonl`.
- Check whether this change creates a new durable convention that belongs in `.codex/spec/`.

## Quality Check

- Verify the implementation against the task acceptance criteria.
- Update relevant spec files when the task changes project behavior, architecture, data contracts, or recurring workflow rules.
- Record session outcome in `.codex/workspace/` before closing substantial work.
