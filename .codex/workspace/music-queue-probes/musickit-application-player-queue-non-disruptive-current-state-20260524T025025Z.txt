## Interpretation

- If Music.app is playing while ApplicationMusicPlayer is stopped with an empty queue, ApplicationMusicPlayer is not evidence of the user's visible Music.app session.
- If ApplicationMusicPlayer has entries, compare them against Music.app's visible Up Next/history UI before any exact claim.
- This probe does not test queue editing. Editing remains locked until a public edit API is proven to modify the same visible Music.app session and the read path proves the result.
