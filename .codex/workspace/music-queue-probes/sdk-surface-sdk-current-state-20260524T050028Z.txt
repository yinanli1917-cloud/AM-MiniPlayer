## Interpretation

- ApplicationMusicPlayer queue availability is not Music.app session parity by itself; visible Up Next/history proof is still required.
- SystemMusicPlayer or MediaPlayer systemMusicPlayer availability would only make a future candidate; visible parity proof is still required before using it.
- MPNowPlayingInfoCenter is current-app now-playing metadata, not a Music.app queue reader.
